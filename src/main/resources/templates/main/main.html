Rest API

Restful 서비스(API)

	- REST 아키텍쳐를 준수하는 서비스
	- Spring MVC Controller에서는 요청 받은 내용을 처리하고 데이터를 가공해서 처리 결과를 "특정 플랫폼"에 적합한 형태의 View를 만들어 응답
	- 하지만 Restful 서비스에서는 데이터만 처리하거나 응답 데이터가 있다면 JSON/XML로 응답함 (View와는 무관)
	=> 클라이언트는 규칙에 맞게 작업을 요청하면 되고 서버는 어떤 플랫폼에서 어떻게 사용할 것인지를 신경쓰지 않고 요청 받은 데이터만 처리하고 응답하면 됨 /
	클라이언트는 받은 데이터를 알아서 가공해서 사용함, 즉 멀티 플랫폼에서 사용 가능.
	이런 특성으로, 사용자 플랫폼과 view에 상관하지 않고 요청한 데이터만 응답해주는 오픈 API에서 Restful한 서비스 방식을 많이 사용함

Rest(Representational Safe Transfer)

	- 데이터를 주고 받는 방식의 아키텍쳐 (디자인 패턴)
	- HTTP URI를 통한 자원의 명시
	* url?category=1 와 같은 쿼리스트링 방식이 아니라 url/category/1과 같은 방식 사용
	* 대문자 사용 지양
	* _ 대신 - 사용
	* URL 마지막에 / 사용하지 않음
	* 파일 확장자 포함하지 않음
	* 행위를 포함하지 않음 (insert/update/delete 등)
	* 가급적 명사 사용 (동사 사용 지양)
	
	- HTTP Method(GET, POST, PUT, DELETE)로 해당 자원을 제어하는 행위를 표현
	* GET - Read(Select) : URL/1
	* POST - Create(Insert) : URL
	* PUT - Update : URL/1
	* DELETE - Delete : URL/1

1. 도서 조회(GET)

	GET book/category/novel
	GET book/category/essay
	GET book/category/poem
	GET book/category/magazine
	GET book/category/dictionary

2. 도서 등록 (POST)

	POST book

3. 도서 수정(PUT)

	GET book/1
	
	PUT book/1

4. 도서 삭제(DELETE)

	DELETE book/1
